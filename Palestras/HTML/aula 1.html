O que é o desenvolvimento WEB?

O desenvolvimento WEB consiste na criação de páginas para a internet, sendo que vamos estar precisando utilizar algumas coisas que podem vir de padrão na maioria dos computadores e navegadores, apenas vamos estar precisando de um elemento que pode acabar não vindo diretamente, que é um editor de códigos.

Nesse mês vamos estar vendo as principais linguagens para fazer isso, lembrando que não são todas de programação, vamos estar separando elas em duas, as linguagens de marcação e as linguagens de programação.

Linguagens de marcação

As linguagens de marcação, elas são mais o que elas mesmo já dizem em seu nome, elas vão estar ali para marcar espaços para o computador entender onde ele vai estar inserindo algum elemento. Como de grande exemplo, em uma página nós temos imagens e parágrafos, a ordenação deles vão ser demarcados por tags no nosso HTML e o estilo no nosso CSS. Sendo que elas vão estar conhecendo de maneira mais completa mais para frente.

Basicamente é apenas isso, Linguagens de marcação são apenas para sinalizar para o computador onde cada um dos elementos ficará!

Linguagem de programação

Agora sim chegamos em um ponto mais interessante, a linguagem de programação, onde vamos estar programando as funções para a nossa página, para isso temos duas linguagens que ainda são líderes no seguimento WEB, que é a JavaScript e o PHP (Hypertext PreProcessor | Personal Home Page), essas duas linguagens vão ser as principais para estarmos conseguindo puxar informações de registros e inserindo funções em botões e elementos da nossa página, deixando assim ela mais funcional e operativa. Além de que assim ela vai se mais útil do que apenas uma página estática.

Análise de desenvolvimento

A Análise de desenvolvimento, é uma parte muito importante para o desenvolvimento de qualquer sistema, seja de computador, mobile ou web, essa etapa faz parte do início de um projeto e programação, onde serão pré planejadas as funções do aplicativo/programa. Igualmente com suas situações de quais objetos serão formados, quais as variáveis/dados serão usados e assim por diante. Não acaba sendo necessário para projetos pequenos grandes diagramas, mas eles podem ser úteis afinal de contas.

Em projetos menores, nós podemos estar utilizando simples listas de afazeres, modelos simples do que vamos estar precisando e depois estar em meio ao desenvolvimento juntando cada vez mais itens e necessidades para o nosso projeto.

As etapas de uma análise cobrem partes de:

  • Atividades: O que o usuário vai estar fazendo com o aplicativo, programa ou site
  • Classes: Classes ou objetos são partes dos elementos com quais vamos estar trabalhando no nosso software, como um item visual, ou um tipo de registro, como um perfil de uma rede social que é formado de Nome, Imagem de Perfil, E-mail e Senha.
  • Casos: Em que vamos estar juntando em uma questão mais descritiva os “ses” do nosso aplicativo, para gerar mensagens de erro, finalização de processo ou inicialização de outros processos, como de exemplo, se o valor de um texto for igual a um determinado, acontecer X caso contrário acontecer Y.
  • Sequencia: Quais funções vão ser disparadas pelo o que, e o que vai estar vindo a seguir. Tendo sempre seu ponto de início e seu ponto final.

    Iniciando o nosso projeto

    Bem, após esclarecer os itens iniciais, nós vamos estar começando agora com os nossos arquivos iniciais, onde vamos estar precisando do arquivo HTML, arquivo CSS e o arquivo JS, o arquivo HTML vamos dar o nome dele de “index”, CSS de “style” e o JS de “script”. Sendo assim teremos 3 arquivos, o “index.html”, o “style.css” e o “script.js”. O porque desses nomes é apenas por serem os nomes mais utilizados e mais comuns para a criação de uma página HTML.

    Para facilitar, o Visual Studio code, ele consegue estar nos ajudando com as primeiras linhas do nosso HTML, então quando estamos com o arquivo index.html, precisamos apenas digitar “html” que o visual studio code já vai nos apresentar algumas das opções para iniciar o nosso arquivo. O que mais estaremos usando vai ser no momento o “HTML:5”, qual já vai estar preparando o nosso arquivo HTML a versão 5 do mesmo.
